Chatbot Inventory Platform: A multi-tenant Chatbot Inventory Platform that allows admins and vendors (companies) to create, configure, and manage RAG-based chatbots. Each vendor will have isolated data, API keys, document stores, and usage analytics.

1. Admin:
Admin Dashboard: Will have a profile icon and profile information-> get admin. Profile will have settings and services. 
->Settings-> update admin profile, change password
->Services-> change vendor status, view uploaded documents, duplicate chatbots, get most users of a vendor, get most chatbots of a vendor, get most used chatbot, total tokens used by vendors, From Embeddings:add embedding,get list of embeddings,get an embedding,update an embedding,delete an embedding, From LLM:create llm,get list of llms, get a llm,update llm,delete llm

endpoints:
->create admin-> @router.post("/create", response_model=AdminRead)
->get admin-> @router.get("/me/{admin_id}", response_model=AdminRead)
->update admin profile->@router.put("/edit/{admin_id}", response_model=AdminRead)
->change password->@router.put("/change-password")
->change vendor status->@router.put("/update-vendors/{vendor_id}", response_model=VendorRead)
->view uploaded documents->@router.get("/documents", response_model=List[DocumentRead])
->duplicate created chatbot->@router.post("/chatbots/duplicate/{chatbot_id}")
->get most users of a vendor->@router.get("/most-users-by-vendors")
->get most chatbots of a vendor->@router.get("/most-chatbots-by-vendors")
->get most used chatbot->@router.get("/most-used-chatbot")
->total tokens used by vendors->@router.get("/total-tokens/{vendor_id}")

6. Document:
Instructions: Vendor dashboard will have an upload document button. Clicking the buttton will open a sub window that has the following fields->  upload document, summary, tags and at the bottom right most corner->save document. When using the upload document field, it will call the preview document endpoint. Preview document will return title, summary, tags that will be ai generated and the summary and tags field will be populated by these values. save document button will use the save document endpoint. This endpoint will save the document in database. 
endpoints:
->preview document-> @router.post("/preview", response_model=DocumentCreate)
->save document->@router.post("/add", response_model=DocumentRead)
->get all uploaded documents->@router.get("/", response_model=List[DocumentRead])
->get a single document->@router.get("/{document_id}", response_model=DocumentRead)
->update a document->@router.put("/{document_id}", response_model=DocumentRead)
->delete a document->@router.delete("/{document_id}")
->create a knowledgebase for rag->@router.post("/{document_id}/knowledge-base")
